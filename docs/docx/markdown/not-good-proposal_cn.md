项目提案：AWS EC2 实例的性能基准测试
性质：技术型

## 引言
如今，随着人们越来越多地意识到云计算的便利和低廉价格，云服务器的日常使用量也在迅速增长。在不同的云服务提供商中，Amazon Web Services (AWS) 是最著名的一个，支持着大多数用户。对于 AWS 来说，Elastic Compute Cloud (EC2) 实例是最受欢迎的服务。因此，从用户的角度来看，在购买 EC2 实例之前，了解不同 EC2 实例在不同环境下的性能非常重要。因此，本项目旨在为用户提供 AWS EC2 实例的详细性能评估。

## 问题陈述
从用户的角度来看，在部署相应工作之前了解不同环境下的不同 EC2 实例性能对于实现利益最大化至关重要。EC2 实例的评估可以分为两个方面：实例属性性能和网络性能。在实例属性性能方面，用户可以从 AWS 官方文档 [1] 中搜索基础性能，或从 Cloudlook [2] 了解不同类型实例的基准测试，其结果基于大量的测量。因此，本项目旨在测量不同类型 EC2 实例的 CPU、内存和 I/O 性能，以评估之前评估的准确性。对于网络性能，一项研究 [3] 提供了提示，不同的工作负载可能会在不同时间部署在不同地区的实例上。因此，用户有必要从这两个维度了解网络性能，这也是本项目将要评估的内容。

## 目标
* 评估不同类型 EC2 实例的 CPU 性能、内存性能和 I/O 性能。
* 评估部署在不同地区的 EC2 实例的 TCP 吞吐量和往返时间 (RTT) 等网络性能，并探索变化规律。
* 评估不同时间的网络性能并探索变化规律。

## 范围
* 对于第一个目标，本项目旨在评估 T2、T3、M4、M5 类型的 EC2 实例。
* 对于第二个目标，本项目旨在评估部署在北弗吉尼亚、俄勒冈、东京、爱尔兰和圣保罗的 EC2 实例。
* 对于第三个目标，本项目旨在评估大约一个月内每天不同时间段的 EC2 实例网络性能，并比较工作日和非工作日的性能。
